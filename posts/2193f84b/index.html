LCOF 322：CLICK ME

给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

解题思路一：
首先，这题回溯一定会超时，因此采用动态规划来进行优化，这里定义两个变量，一个是选用硬币的序号，另一个是总金额。
定义 dp 数组，里面的元素 dp[j] 表示要达到总金额 j 时最少使用的硬币数量。
假设使用下标为 0 ~ i-1 那一行的最少硬币数量已经确定，此时需要更新第 i 行的最少硬币数量，第 i 个硬币有放或者不放两个选项，我们选其中更少的那个更新 dp[j]，有 dp[j] = std::min(dp[j], dp[j - coins[i]] + 1)，不难看出 dp[j] 的值其实也是依赖它的上一个状态，也就是 dp[j - coins[i]] 的，所以最后 dp 数组会更新 coins.size() * amounts 次。

解题思路二：
优化回溯，本质是 DFS + 剪枝。
首先先对硬币大小排序，如果要硬币数量尽可能的少，当然要先用大面额的硬币，这里需要注意，不能用贪心，因为贪心只是局部最优，我们要取得全局最优，只能把所有的情况都递归完。

LeetCode 3：CLICK ME

给定一个字符串，请你找出其中不含有重复字符的最长子串的长度。

解题思路：
这种类型的题目有一种类似的套路，对于字符串中下标为 i 的字符 str[i]，我们去寻找以该字符结尾的最长不重复子串有多长，当判断到最后一个字符时，也就找到了答案，时间复杂度一般都为 O(n)。
本题中，我们需要使用一个哈希表和一个变量 pre，哈希表中存储的是某一个字符在字符串中最近一次出现的下标，变量 pre 表示必须以 str[i-1] 结尾的情况下，最长不重复子串的起始下标的前一个下标，即 pre+1 ~ i-1 这个子串是不重复的。
此时来看 str[i] 有两种情况，第一种 pre+1 ~ i-1 中有 str[i]，也就是哈希表中的 map[str[i]] 大于 pre，此时的不重复子串范围为 map[str[i]] ~ i，需要把 pre 更新为 map[str[i]]，而如果 map[str[i]] 小于 pre，不重复子串范围为 pre ~ i。

LeetCode 4：CLICK ME

给定两个大小为 m 和 n 的有序数组 nums1 和 nums2。

请你找出这两个有序数组的中位数，并且要求算法的时间复杂度为 O(log(m + n))。

你可以假设 nums1 和 nums2 不会同时为空。

解题思路：
看到对数时间复杂度，应该想到利用二分的方式查找中位数。
首先，在任一位置 i 将长度为 m 的数组 A 划分成两个部分：
leftA | rightA
A[0],A[1],… A[i-1] | A[i],A[i+1],…A[m - 1]
由于 A 有 m 个元素，所以有 m + 1 种划分方式，len(leftA) = i，len(rightA) = m - i
采用同样的方式，将 B 也划分为两部分：
leftB | rightB
B[0],B[1],… B[j-1] | B[j],B[j+1],…B[n - 1]
同样的，len(leftB) = j，len(rightB) = n - j

将 leftA 和 leftB 放入一个集合，将 rightA 和 rightB 放入一个集合。再把这两个集合分别命名为 leftPart 和 rightPart。
leftPart | rightPart
A[0],A[1],… A[i-1] | A[i],A[i+1],…A[m - 1]
B[0],B[1],… B[j-1] | B[j],B[j+1],…B[n - 1]
当左右两边的长度大致相等的时候，划分效率是最高的，也就是说，如果满足下面两个条件：
len(leftPart) = len(rightPart); // m + n 为奇数时不成立
max(leftPart) <= min(rightPart); // 左半边的最大值小于右半边的最小值
就能够推断出：
median = (max(leftPart) + min(rightPart))/2
因为此时相当于一个数组左右两边数量相等，中位数就是位于中间两个数和的平均数

如果要满足上面的假设，需要确保这两个条件满足：
i + j = 偶数时：m - i + n - j 奇数时：m - i + n - j + 1
为了保证 j 不为负数，我们可以假设 n >= m，只需要使 i = 0 ~ m，j = (m + n + 1)/2 - i 此时奇偶都成立
B[j] >= A[i-1] 并且 A[i] >= B[j-1]，因为它们原数组的大小已经得到了保证

此外还要注意临界条件：i = 0，j = 0，i = m，j = n。

所以我们可以按照以下步骤进行二叉搜索
设 imin = 0，imax = m，然后开始在 [imin, imax] 中进行搜索
令 i = (imin + imax)/2，j = (m + n + 1)/2 - i
现在我们有 len(leftPart) = len(rightPart)。而我们只会遇到三种情况：
①.B[j] >= A[i-1] 并且 A[i] >= B[j-1] 满足条件
②.B[j-1] > A[i]。此时应该把i增大。 即imin = i + 1;
③.A[i-1] > B[j]。此时应该把i减小。 即imax = i - 1;

整个调整过程其实是不断地增大 i 减小 j 或者增加 j 减小 i，始终 leftPart 和 rightPart 数量相等，然后找到一个使得左边最大值小于右边最小值的分界线。
