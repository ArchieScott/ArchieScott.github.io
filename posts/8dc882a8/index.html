LeetCode 0392. 判断子序列
发表于 2020-07-27| 更新于 2020-07-29|LeetCode
字数总计:831|阅读时长: 3 分钟
|阅读量:

题目

题目链接
给定字符串 s 和 t，判断 s 是否为 t 的子序列。
你可以认为 s 和 t 中仅包含英文小写字母。字符串 t 可能会很长（长度 ~= 500,000），而 s 是个短字符串（长度 <= 100）。
字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，“ace” 是 “abcde” 的一个子序列，而 “aec” 不是）。

示例 1:
s = “abc”, t = “ahbgdc”
返回 true.

示例 2:
s = “axc”, t = “ahbgdc”
返回 false.

后续挑战 :
如果有大量输入的 S，称作 S1, S2, … , Sk 其中 k >= 10 亿，你需要依次检查它们是否为 T 的子序列。在这种情况下，你会怎样改变代码？

思路一：双指针

一种比较容易想到的思路就是双指针，我们用两个指针 i 和 j 分别指向字符串 s 和 t，每当匹配成功的时候，i 和 j 都向后移动一个字符，如果匹配失败，j 向后移动一个字符，i 不动，当 i 移动到末尾都能匹配成功返回 true，否则返回 false。

思路二：动态规划

动态规划主要用来解决的是字符串 s 的数量特别多的情况，如果一条一条依次匹配，会有大量的时间花在移动不匹配的字符指针上，那么我们其实可以处理出这样的信息：
对于被匹配串 t 的每一个位置，可以用一个 dp 数组来表示对应位置分别到 26 个字符中下个字符的距离，这样我们只需要查 dp 数组就能得到字符串 s 的下一个字符在 t 的哪一个位置。
那么 dp[i][j] 表示字符串 t 的第 i 个字符后第一次出现字符 j 的位置，如果 t[i] == j，那么 dp[i][j] = i，否则当 j 出现在 i 之后，有 dp[i][j] = dp[i + 1][j]，变为一个由后向前递推的动态规划。
因为 i 的取值范围为 0 到 m - 1，而 dp[i][j] 是每一个字符 j 都在 dp 数组中，因此 dp[i][m] 一定能在 dp 数组中找到，所以如果 s[i] 的下一个对应字符位置已经到了 m，说明已经匹配不到了，返回 false。
